Camadas 2 e 3 do Modelo OSI: Diferenças e Protocolos de Endereçamento

O modelo OSI (Open Systems Interconnection) define as camadas que mantém a estrutura entre sistemas em uma rede, para que as transmissões de dados entre o emissor e o receptor sejam padronizadas. A seguir irei detalhar um pouco sobre as camadas 2 e 3.

Camada 2 - Enlace ou ligação

Basicamente, esta camada verifica se as requisições têm algum defeito, controlando os erros com que frequências são enviados, ajudando para que as camadas superiores possam fazer uma transmissão praticamente sem erros e de forma mais confiável.

Camada 3 - Rede

Ela é a camada que mantém a comunicação entre os emissores e receptores, controlando como os dados são transferidos e garantindo que cheguem ao destino da melhor forma possível.

Enquanto a camada de enlace trabalha com endereço MAC (este endereço é unico para cada dispositivo) e conecta dispositivos da mesma rede, a camada da rede conecta dispositivos de redes diferentes trabalhando com endereços de IP(este endereço é mantido, para anotar a origem e o destino da requisição).
A camada 2 utiliza protocolos como o Ethernet (IEEE 802.3), muito comum entre redes cabeadas, WIFI (IEEE 802.11) padrão de redes sem fio e HDLC, comum em comunicações WAN. Já a camada 3 utiliza alguns como o IPv4, versão mais antiga mas ainda amplamente utilizada e o IPv6, sucessor do IPv4 pois suporta um número bem maior de endereços.

Soluções Proprietárias vs. Open Source

Ambas são boas opções mas com diferentes pros e contras:
Enquanto soluções proprietárias como o Windows contam com um suporte oficial de uma empresa e suas parceiras, uma maior popularidade e uma maior compatibilidade com softwares populáres, já que muitos ainda não fazem uma versão para o "concorrente", ele oferece desvantagens como maior concetração de ataques te malwares (devido a maior popularidade), falta de flexibilidade no sistema já que seu código fonte "não pode" ser alterado e um custo elevado.
Soluções Open Source como o Ubuntu, Debian e outros sistemas operacionais baseados em Linux oferecem uma maior liberdade, você pode fazer praticamente o que quiser no sistema, inclusive fazer ele parar de funcionar, o seu uso é gratuito e por ser menos popular tem uma faixa menor de malwares dedicados a ele, suas desvantagens são a maior curva de aprendizado, já que muitas coisas precisam de um conhecimento técnico maior para a utilização de alguns recursos, seu suporte pode ser mais dificil de encontrar, já que ele não tem uma empresa responsável para isso, dependendo de foruns e videos no YT, e conseguir fazer com que alguns aplicativos populares rodem no sistema pode ser um problema, já que as empresas focam mais em produzir para os sistemas mais populares como o Windows.

Open Source o que é?

Open Source significa código aberto, ou seja, o código-fonte de um software está disponível publicamente, permitindo que qualquer pessoa possa sugerir alterações, melhorias ou até mesmo propor mudanças no projeto. Exemplos conhecidos de software open source incluem o OpenJDK (uma distribuição open source do Java) e o Linux (um kernel de código aberto amplamente utilizado em sistemas operacionais e sistemas embarcados).
Muitas grandes empresas, como Google, Microsoft e Red Hat, contribuem para projetos open source, seja com recursos financeiros ou contratando desenvolvedores para se dedicarem exclusivamente à melhoria dessas soluções de código aberto. Esse apoio pode resultar em suporte mais robusto, estabilidade aprimorada e novos recursos para o software. Empresas como a Red Hat, por exemplo, oferecem um sistema baseado no Linux, mas cobram pelo suporte e pela garantia de seus produtos, garantindo assim a continuidade e estabilidade da solução, mas com custos adicionais para os usuários.
Porém, uma desvantagem desse modelo é que as empresas podem tentar direcionar o desenvolvimento do software open source para atender a suas próprias necessidades comerciais, o que pode, em certos casos, atrapalhar o desenvolvimento natural e a evolução do projeto, afastando-o de seu propósito original ou dos interesses da comunidade.
